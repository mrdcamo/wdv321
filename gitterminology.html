Understanding Git Terms

Version Control Software

Version control software helps developers keep track of changes in their code over time. It allows multiple people to work on the same project without messing up each other’s changes. If something goes wrong, you can always go back to a previous version. Git is one of the most popular tools for this.

Add

When you make changes to files in Git, they don’t get saved automatically. The git add command stages those changes so they’re ready to be committed. Think of it as putting files in a box before sealing it up with a commit.

Commit

A commit is like taking a snapshot of your project at a certain point in time. It saves all the staged changes with a message explaining what was updated. This helps keep track of progress and makes it easy to roll back if needed. You commit changes using git commit -m "your message".

Push

Pushing sends your commits from your local computer to a remote repository, like GitHub or GitLab, so others can see and work with your updates. The command git push origin main (or another branch name) does the trick.

Pull

Pulling updates your local files with the latest changes from a remote repository. This is important when working in a team, so you always have the most up-to-date version of the project. You can do this with git pull.

Clone

Cloning is like downloading a full copy of a project from a remote repository to your computer. This is how you start working on an existing project stored somewhere like GitHub. You use git clone <repository URL> to get a copy.
